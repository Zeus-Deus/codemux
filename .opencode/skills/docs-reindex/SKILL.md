# Docs Reindex

## When to use me

Use this when the docs system has drifted and needs a fresh cleanup pass from actual repo evidence.

## What I do

1. Read `WORKFLOW.md` and `docs/INDEX.md`
2. Read `docs/reference/DOCS_REINDEX.md`
3. Audit the real repo before trusting old docs
4. Update canonical docs from code reality
5. Remove or archive stale duplicates once clean replacements exist
